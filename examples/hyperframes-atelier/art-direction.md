# Art Direction — OpenMontage: "Code Into Film"

Bespoke HyperFrames atelier composition. Hand-authored; no stock `cut.type` scenes.

## Design read
An **editing bay / director's terminal** where written instructions become film.
The piece literally shows code turning into motion — the product's thesis.

## Signature device
The **clapperboard slate stripes** (Backlot identity). Appears only in the cold
open (assembling) and returns transformed as the closing underline. Scarce by design.

## Palette (unique to this piece)
- bg graphite `#0B0D12`, panel ink `#12151C`
- bone `#EDEBE3`, muted `#5B6472`
- slate amber `#F2A900` (the slate), render cyan `#37E0C8` (the "render" signal)

## Type
- Display: Space Grotesk (tight grotesk)
- Code: JetBrains Mono

## Motion language (≥3 easings, entrances AND exits)
`power4.out` entrances · `expo.inOut` moves · `back.out(1.7)` pops ·
`sine.inOut` pulse · `power2.in` exits. Each scene has a distinct motion register.

## Scenes (distinct primary subject / composition / scale each)
1. Cold open — clapperboard slate claps, wordmark snaps in (centered, bold).
2. Code → motion — split: `edit_decisions` code types left, transforms into moving bars right (busy/technical).
3. Pipeline rail — 7 stage nodes ignite in sequence with a travelling cyan pulse (wide, low-third).
4. Runtime choice — three runtime chips rise; HyperFrames gets an accent ring (grid, accent choreography).
5. Closing lockup — slate stripes return as an underline under "code → video, on your machine" (intimate), then exits.

## Distinctness check
Could this be any other product's video? No — the slate signature, the literal
`edit_decisions`→bars transform, and the named pipeline stages are OpenMontage-specific.
No two scenes share a primary visual subject.
